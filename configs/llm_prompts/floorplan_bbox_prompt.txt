이 이미지는 크롤링된 건축 평면도 데이터셋에서 가져온 DXF 도면의 렌더링이다.
대부분 평면도이며, 간혹 여러 도면이 섞여 있거나 다른 종류의 도면일 수 있다.

작업:
- 이미지에 건축 평면도(floor plan)가 있는지 확인하고, 있다면 각 평면도의 위치를 bbox로 반환하라.

평면도의 특징 (이 중 하나라도 해당되면 평면도로 판단):
- 벽체를 나타내는 닫힌 다각형 또는 연결된 선들
- 위에서 내려다본 방 배치 (방/복도/주방 등의 구획)
- 문의 호(arc), 창문 표시, 가구 심볼
- 치수선과 방 이름 라벨

평면도가 아닌 것:
- 단면도/입면도 (수직 방향 뷰)
- 3D 투시도, 렌더링 이미지
- 순수한 표/차트/리스트
- 타이틀블록만 있는 경우

중요 지침:
1. **평면도로 보이면 관대하게 판단하라.** 확실하지 않아도 평면도 특징이 조금이라도 보이면 포함한다.
2. 이미지 전체가 하나의 평면도면 bbox를 거의 전체 영역으로 설정 (예: 0.0~1.0).
3. 층별로 여러 평면도가 있으면 각각 별도 bbox로 반환.
4. **좌표는 반드시 0.0 이상 1.0 이하의 실수 (정규화 좌표).** 픽셀 값 금지.
   - 올바른 예: "x_min": 0.12, "x_max": 0.88
   - 잘못된 예: "x_min": 115, "x_max": 735 (이미지 픽셀 좌표는 금지)
5. x_min < x_max, y_min < y_max 반드시 성립.
6. 이미지 좌상단이 (0, 0), 우하단이 (1, 1)이다.

출력은 반드시 JSON 객체 하나만. 다른 텍스트/마크다운/코드블록 금지.

형식:
{
  "floorplans_found": true,
  "floorplans": [
    {
      "label": "floorplan_0",
      "reason": "벽체와 방 구획이 명확함",
      "bbox": {
        "x_min": 0.05,
        "y_min": 0.05,
        "x_max": 0.95,
        "y_max": 0.95
      }
    }
  ]
}

평면도가 전혀 없으면:
{"floorplans_found": false, "floorplans": []}

추가 작업:
- 각 평면도가 몇 층인지 추론하여 "label" 필드에 답하라.
  • "B1" "B2" — 지하층 (basement)
  • "1F" "2F" "3F" ... — 지상층
  • "RF" — 옥상/지붕층 (roof)
- 추론 단서: 도면 내 "1층", "2F", "지하1층" 같은 한글/영문 라벨, 평면도 위치
  (보통 책상에 펼친 도면처럼 1층이 좌하단/하단, 위층이 위쪽)
- 추론 어려우면 "label" 을 "floorplan_N" 으로 표시 (N = 0부터)

출력 형식 변경 — 각 floorplan 에 floor_index 필드 추가:
{
  "floorplans_found": true,
  "floorplans": [
    {
      "label": "1F",
      "floor_index": 0,
      "reason": "...",
      "bbox": {...}
    }
  ]
}
floor_index 규칙:
- 1층 = 0, 2층 = 1, 3층 = 2 ... (0-based)
- 지하1층(B1) = -1, 지하2층(B2) = -2
- 옥상(RF) = 가장 큰 지상층 + 1
- 추론 실패 시 floor_index = -999
